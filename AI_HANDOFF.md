# AIへの依頼書

このファイルを、サイトの保守を依頼するAIへ最初に渡してください。

特定のAI製品だけに依存しません。Work、一般的なチャットAI、コーディングAI、スマートフォンから利用するAIでも、GitHubの読み書き権限があれば同じ方針で作業します。

## 依頼

鈴子ちゃん公式サイトを、現在の見た目、文章、画像、軽量な静的サイト構成を維持しながら保守してください。

提案だけで止まらず、利用できる権限とツールの範囲で、調査、変更、検査、GitHubへの反映、公開確認、結果報告まで実行してください。

公式サイト

https://abcderp2.github.io/suzukochan.officialsite/

正本リポジトリ

https://github.com/abcderp2/suzukochan.officialsite

公開元はmainブランチです。

## 最優先方針

現在正常に動作している公開サイトを大きく作り直しません。

依頼された目的に必要な最小限のファイルと行だけを変更します。依頼と関係のない整形、言い換え、順番変更、最適化、リファクタリングは行いません。

追加課金が必要なサービス、有料API、外部ホスティング、外部CMS、外部CDN、アクセス解析、広告、外部フォームは追加しません。

Node.js、npm、フレームワーク、ビルドシステム、データベース、バックエンドは追加しません。Pythonを使う場合は標準ライブラリだけを使います。

パスワード、個人アクセストークン、APIキー、二段階認証コード、復旧コード、秘密鍵、メール認証コードを要求、記録、表示、コミットしません。

## 自動で変更してよい範囲

権利者から作業依頼がある場合、次の保守作業はAIが変更まで実行できます。

- 保守資料とチェックリストの更新
- 404ページなどの補助ページの修正
- 存在しないローカル参照の修正
- 明らかなURL誤記の修正
- alt属性やnoopenerの不足など、意味を変えない機械的修正
- Markdown、JSON、YAML、HTMLの単純な構文不良の修正
- 読み取り専用の自動チェックの改善
- GitHub Actionsの読み取り専用検査の改善

## 明示的な承認が必要な範囲

次は、権利者が変更対象と内容を明示した場合だけ変更します。不明な場合は変更せず、確認事項として報告します。

- 鈴子ちゃんの設定、プロフィール、本文の意味
- 利用ガイドライン
- LICENSEと権利表記
- 画像の追加、削除、差し替え、加工、変換、圧縮、リサイズ、ファイル名変更
- デザイン、色、レイアウト、CSS
- robots.txt、sitemap.xml
- GitHub Pages設定、リポジトリ設定、ブランチ保護
- 外部公開先や公式URL

## 標準の自動作業手順

1. 最新のmainと未処理のプルリクエストを確認します。
2. 変更前のmainを指す復旧用ブランチを作成します。
3. 最新のmainから一時作業ブランチを作成します。
4. 依頼された範囲だけを変更します。
5. `python3 scripts/check_site.py`を実行します。
6. 差分を確認し、依頼外の変更がないことを確認します。
7. プルリクエストを作成します。
8. 自動チェックが成功したら、Squash and mergeでmainへ一つのコミットとして反映します。
9. GitHub Pagesの公開と公式サイトを確認します。
10. 変更内容、検査結果、mainのコミット、復旧用ブランチ、戻し方を報告します。

一時作業ブランチの複数コミットは、mainでは一つのSquashコミットにまとめます。これにより、一回のRevertで変更全体を戻せます。

AIの環境がプルリクエストを作れず、権利者からmainへの直接反映を明示されている場合は、変更前の復旧用ブランチを作成し、事前検査に成功した変更を一つのコミットとしてmainへ反映します。

## ブランチ名

復旧用

`backup/YYYYMMDD-HHMM-目的`

一時作業用

`work/YYYYMMDD-HHMM-目的`

既存ブランチを上書きしません。

## 失敗時の処理

事前検査に失敗した変更はmainへ反映しません。同じ作業ブランチで修正して再検査します。

mainへの反映後に検査または公開確認が失敗した場合は、対象のSquashコミットをRevertするコミットを作成し、再度検査と公開確認を行います。

force push、hard reset、履歴の削除は行いません。復旧用ブランチでmainを強制上書きしません。

Revertが競合する場合は、復旧用ブランチと現在のmainを比較し、対象作業の変更だけを打ち消す新しいコミットを作成します。

## スマートフォンだけで作業する場合

GitHubアプリまたはスマートフォンのブラウザだけでも保守できます。

AIにこのファイルと依頼内容を渡し、作成されたプルリクエストで次を確認します。

- Files changedが依頼したファイルだけになっている
- ChecksまたはActionsが成功している
- mainへの反映が一つのSquashコミットになっている
- 反映後に公式サイトが表示される

問題がある場合は対象コミットまたはプルリクエストのRevertを使います。

## 検査

`python3 scripts/check_site.py`

検査スクリプトは報告だけを行い、ファイルの自動修正、コミット、マージは行いません。修正は依頼を受けたAIが差分を確認しながら実行します。

## 完了報告

- 変更したファイル
- 変更理由
- 実行した検査と結果
- mainへ反映したコミット
- 復旧用ブランチ
- 公開確認の結果
- 未解決事項
- Revert対象
